About

QA Engineer - manual-first - process-driven - results-proven across 7 industries

My QA practice is built on a structured manual testing foundation: requirement traceability, risk-based prioritisation, and exploratory techniques applied systematically before any automation is considered. This discipline ensures coverage is deliberate, not accidental.

I introduce automation purposefully - through hybrid test frameworks combining Playwright and Cypress - targeting repetitive regression cycles and high-risk product areas. Frameworks follow OOP principles: Page Object Model architecture with class-based structures, encapsulation, and inheritance for long-term maintainability. BDD/Cucumber keeps specifications readable across business and engineering teams.

Specialist disciplines: GDPR compliance testing, WCAG 2.1 accessibility validation, mobile (iOS/Android) functional testing, and AI Agentic Testing using Claude AI. I own the QA process from requirement review through release sign-off - including RTM maintenance, defect triaging, severity classification, and weekly stakeholder reporting.
